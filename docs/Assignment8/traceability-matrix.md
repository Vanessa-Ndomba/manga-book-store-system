# Traceability to Functional Requirements

## The diagrams in this assignment align with functional requirements developed earlier. For example:

- User Account diagrams support user registration and login requirements.
- Shopping Cart and Checkout workflows support cart management and order placement requirements.
- Payment and Return diagrams support transaction handling and refund requirements.
- Shipment diagrams support delivery tracking requirements.
- Review diagrams support customer feedback functionality.

  
## Traceability to User Stories and Sprint Tasks
These diagrams also support Agile planning by clarifying the behavior behind user stories such as:

- As a customer, I want to register an account so that I can place orders.
- As a customer, I want to browse manga so that I can find books to buy.
- As a customer, I want to add books to a cart so that I can purchase multiple items.
- As an admin, I want to update inventory so that the catalog remains accurate.
- As a customer, I want to request a refund so that I am protected when a product is faulty.
- The activity diagrams help translate those stories into implementable flows, while the state diagrams clarify how important business objects behave over time.
